## Connector SDK

This document describes the FlowFoundryPro Connector SDK and examples for Slack and Gmail typed clients.

### Slack Client

Example usage with an incoming webhook URL (dev/demo):

```ts
import { createSlackClient } from "@flowfoundry/connectors";

const slack = createSlackClient({
  orgId: "demo-org",
  credentials: { webhookUrl: process.env.SLACK_WEBHOOK_URL! },
  emitLog: (msg, extra) => console.log(msg, extra)
});

await slack.sendMessage({ text: "Hello from FlowFoundryPro" });
```

### Gmail Client (Demo)

The Gmail client validates inputs and enqueues a demo result:

```ts
import { createGmailClient } from "@flowfoundry/connectors";

const gmail = createGmailClient({ orgId: "demo-org", emitLog: () => {} });
await gmail.sendEmail({ to: ["user@example.com"], subject: "Hi", html: "<b>Hello</b>" });
```

### Inngest Slack Send Function

The app includes an Inngest function `slack-send-message` that posts with retry/backoff.
Emit event `slack.send` with data shape `{ orgId, text, webhookUrl? | apiKey? }`.

### Credential Storage (Dev Mock)

Use the tRPC endpoint to upsert a base64-encoded JSON envelope (dev mock):

```ts
// Base64 of { "webhookUrl": "https://hooks.slack.com/..." }
const envelopeB64 = Buffer.from(JSON.stringify({ webhookUrl: process.env.SLACK_WEBHOOK_URL })).toString("base64");
await fetch("/api/trpc/connectors.upsertCredential", { method: "POST", body: JSON.stringify({ connectorId: `${orgId}_slack`, envelopeB64 }) });
```


# Connector SDK

```ts
export type Connector = {
  id: string;
  name: string;
  auth: "oauth2" | "apiKey" | "none";
  actions: Record<string, (input: unknown, ctx: ConnectorCtx) => Promise<unknown>>;
};
```
Built-ins: HTTP, Slack (webhook), Notion (mock), Sheets (mock), Gmail (mock), Webhook.
